# 개발 로그 - 2026-02-01

## 📋 개요
PPT 번역기의 **형광펜(배경색) 번짐 문제** 분석 및 해결 과정을 정리한 문서입니다.

---

## 🔍 문제 분석

### 증상
번역 후 형광펜이 원본보다 훨씬 넓은 범위에 적용되는 **"번짐(Leaking)"** 현상 발생

### 수치 분석
| 구분 | 원본 | 번역 결과 | 변화율 |
|------|------|-----------|--------|
| 형광펜 | 18자 (10.7%) | 365자 (78.3%) | **2028%** (20배) |
| 글자색 | 175자 | 784자 | 448% (4.5배) |

### 원인
1. **AI(Gemini Flash)의 한계**: 한→영 번역 시 어순이 바뀌면 태그 위치를 정확히 추적하지 못함
2. **Run 파편화**: 원본 PPT에서 형광펜이 여러 개의 분리된 Run으로 나뉘어 있음
   - 예: `['종사자', '의 생명과 신체를 보호함을 ']` (2개 조각)

---

## 🛠️ 시도한 해결책

### 1차 시도: Run Merging (실패)
- **접근**: 연속된 동일 스타일 Run을 병합하여 태그 파편화 방지
- **결과**: 여전히 형광펜 번짐 발생

### 2차 시도: Safety Mode (실패)
- **접근**: 번역 후 형광펜 비율이 원본의 2배 이상이면 자동 제거
- **결과**: 조건이 제대로 작동하지 않음

### 최종 결정: 형광펜 기능 완전 제거 ✅
- **이유**: 복잡한 수정보다 깔끔하게 기능 제거가 더 안정적
- **영향**: 원본 PPT의 형광펜은 무시되고, 번역 결과에 형광펜 미적용

## 📚 교훈 (Lessons Learned)

1. **AI 태그 추적의 한계**: LLM은 어순 변경 시 태그 위치 추적에 취약함
