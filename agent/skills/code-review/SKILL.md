# 코드 리뷰 전문가 스킬
 
이 스킬이 활성화되면, 에이전트는 단순한 코드 설명을 넘어 다음 기준에 따라 엄격하게 리뷰를 수행합니다.
 
## 🕵️ 리뷰 체크리스트
 
1. **논리적 결함 (Correctness)**
   - 코드가 의도한 대로 동작하는가?
   - 경계값(Edge cases) 처리가 되어 있는가? (null check, 빈 배열 등)
 
2. **보안 (Security)**
   - 하드코딩된 비밀번호나 API 키가 있는가?
   - 인젝션 공격이나 데이터 노출 위험이 없는가?
 
3. **가독성 및 유지보수 (Clean Code)**
   - 변수와 함수 이름이 그 역할을 명확히 설명하는가?
   - 함수의 크기가 너무 크지 않은가? (단일 책임 원칙)
   - 중복된 로직(DRY 원칙)이 존재하지 않는가?
 
4. **성능 (Performance)**
   - 불필요한 루프나 무거운 연산이 반복되고 있지 않은가?
   - 메모리 누수 가능성이 있는가?
 
##  피드백 가이드라인
 
- **Constructive**: 단순히 "나빠요"가 아니라, "이런 이유로 이렇게 바꾸는 게 어떨까요?"라고 제안합니다.
- **Priority**: 수정이 필수적인 사항(Critical)과 권장 사항(Nitpick)을 구분합니다.
- **Example**: 개선된 코드 예시를 반드시 함께 제공합니다.
 
## 실행 방법
 
주인님이 코드를 보여주며 "리뷰해줘"라고 하거나, 제가 판단하기에 코드 개선이 필요해 보이면 이 지침에 따라 답변하겠습니다.
